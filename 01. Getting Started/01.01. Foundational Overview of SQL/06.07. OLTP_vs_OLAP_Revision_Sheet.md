# 📖 Important Definitions

## ⚡ OLTP

**Online Transaction Processing**

A database workload optimized for handling **large numbers of small, fast, reliable transactions**.

Examples:

* Banking
* Amazon Orders
* Uber Booking
* College Attendance

---

## 📊 OLAP

**Online Analytical Processing**

A database workload optimized for **large analytical queries** over historical data.

Examples:

* Sales Reports
* Revenue Trends
* Customer Analytics
* Business Dashboards

---

## 🏢 Data Warehouse

A centralized repository that stores **integrated historical data** from multiple operational systems for reporting and analytics.

---

## 🔄 ETL

**Extract → Transform → Load**

Transform data **before** loading it into the warehouse.

---

## 🔄 ELT

**Extract → Load → Transform**

Load raw data first and transform it **inside** the warehouse.

---

## ⭐ Star Schema

A data warehouse design where a central **Fact Table** connects directly to multiple **Dimension Tables**.

---

## ❄️ Snowflake Schema

A normalized version of the Star Schema where Dimension Tables are split into additional related tables.

---

# ⚖️ OLTP vs OLAP Comparison

| Feature       | ⚡ OLTP                             | 📊 OLAP                                    |
| ------------- | ---------------------------------- | ------------------------------------------ |
| Purpose       | Run daily business operations      | Analyze historical business data           |
| Users         | Customers, Employees, Applications | Analysts, Managers, Executives             |
| Query Type    | Small, simple queries              | Large, complex queries                     |
| Data          | Current / Operational              | Historical                                 |
| Read/Write    | Frequent reads and writes          | Mostly reads                               |
| Transactions  | Thousands to millions per day      | Comparatively fewer                        |
| Response Time | Milliseconds                       | Seconds to minutes (depending on workload) |
| Data Size     | Smaller operational datasets       | Very large analytical datasets             |
| Database      | Production Database                | Data Warehouse                             |
| Example       | Place Order                        | Monthly Sales Report                       |

---

# 🔄 ETL vs ELT Comparison

| Feature              | ETL                        | ELT                        |
| -------------------- | -------------------------- | -------------------------- |
| Full Form            | Extract → Transform → Load | Extract → Load → Transform |
| Transformation       | Before Loading             | After Loading              |
| Processing           | ETL Tool                   | Data Warehouse             |
| Warehouse Stores     | Processed Data             | Raw + Processed Data       |
| Traditional Systems  | Common                     | Less common                |
| Cloud Data Platforms | Supported                  | Very common                |

---

# ⭐ Star vs ❄️ Snowflake Comparison

| Feature           | Star Schema    | Snowflake Schema |
| ----------------- | -------------- | ---------------- |
| Dimension Tables  | Denormalized   | Normalized       |
| Joins             | Fewer          | More             |
| Query Performance | Usually Faster | Usually Slower   |
| Storage           | More           | Less             |
| Complexity        | Lower          | Higher           |
| Maintenance       | Simpler        | More Structured  |

# 🎤 FAANG Interview Questions

## ❓ What is the difference between OLTP and OLAP?

### ✅ Answer

OLTP is designed for real-time transactional processing with many small, fast operations. OLAP is designed for analyzing large volumes of historical data to support reporting and decision-making.

---

## ❓ Why do companies use both OLTP and OLAP?

### ✅ Answer

OLTP handles day-to-day business operations efficiently, while OLAP supports complex analytical queries. Separating these workloads ensures that reporting does not slow down customer-facing applications.

---

## ❓ Why is a Data Warehouse needed?

### ✅ Answer

A Data Warehouse stores integrated historical data from multiple operational systems. It allows organizations to perform reporting and analytics without affecting production databases.

---

## ❓ What is the difference between ETL and ELT?

### ✅ Answer

In ETL, data is transformed before loading into the warehouse.

In ELT, raw data is loaded first, and transformation occurs inside the warehouse.

---

## ❓ Why is Star Schema generally faster than Snowflake Schema?

### ✅ Answer

Star Schema usually requires fewer joins because Dimension Tables are denormalized. This often improves analytical query performance. Snowflake Schema reduces redundancy but typically introduces additional joins.

# ❌ Common Interview Mistakes

### ❌ Mistake 1

"OLTP stores historical data."

✅ Correct:

OLTP primarily stores operational data for current business activities.

---

### ❌ Mistake 2

"OLAP performs transactions."

✅ Correct:

OLAP is designed for analytics, not operational transactions.

---

### ❌ Mistake 3

"ETL and ELT are the same."

✅ Correct:

The order of transformation and loading is different.

---

### ❌ Mistake 4

"Data Warehouse replaces OLTP."

✅ Correct:

A Data Warehouse complements OLTP; it does not replace it.

---

### ❌ Mistake 5

"Star Schema is always better."

✅ Correct:

Star Schema and Snowflake Schema each have trade-offs. The choice depends on reporting requirements, maintenance needs, and data complexity.
