# 🌍 Introduction

Imagine Amazon.

Every second:

🛒 Orders are placed.

💳 Payments are processed.

🚚 Deliveries are updated.

⭐ Reviews are submitted.

📦 Inventory changes.

All this information is stored in operational databases.

Now imagine the CEO asks:

> "Show the total sales for the last five years."

The operational databases are not designed for this kind of analysis.

So how does the analytical system get the required data?

It cannot simply copy everything without preparation.

The data must first be:

* Extracted
* Cleaned
* Converted into a common format
* Loaded into the Data Warehouse

This entire pipeline is called **ETL**.

---

# 📖 What is ETL?

## Definition

**ETL** stands for:

* **E** → Extract
* **T** → Transform
* **L** → Load

It is the process of collecting data from one or more sources, cleaning and transforming it, and then loading it into a Data Warehouse for analysis.

---

# 🏠 Real-Life Analogy

Imagine you're cooking dinner.

Step 1

🥕 Buy vegetables.

↓

Extract

---

Step 2

🔪 Wash and cut them.

↓

Transform

---

Step 3

🍳 Cook and serve.

↓

Load

Only after these steps is the meal ready.

ETL follows the same idea.

# 📤 Step 1 — Extract

## 📖 Definition

Extraction means **collecting data from different source systems**.

Sources may include:

* SQL Databases
* NoSQL Databases
* CSV Files
* Excel Files
* APIs
* CRM Systems
* ERP Systems
* Log Files

---

# 🌍 Example

Suppose Amazon has:

```text
Orders Database

Payments Database

Customers Database

Inventory Database
```

Each stores different information.

The ETL process first extracts data from each source.

---

# 🤔 Why Can't We Skip Extraction?

Because data is scattered.

Example:

Customer information

↓

Customer Database

Payment information

↓

Payment Database

Inventory

↓

Inventory Database

To analyze the complete business,

all these datasets must first be collected.

---

# 🔄 Step 2 — Transform

This is the most important stage.

## 📖 Definition

Transformation means **cleaning, validating, standardizing, and preparing the data before storing it in the warehouse**.

Think of it as improving raw data so that it becomes useful.

---

# 🧹 Example 1 — Remove Duplicates

Suppose the customer database contains:

| ID  | Name  |
| --- | ----- |
| 101 | Rahul |
| 101 | Rahul |

Duplicate records should usually be removed.

---

# 📝 Example 2 — Standardize Formats

Suppose one system stores:

```text
12-07-2026
```

Another stores:

```text
2026/07/12
```

The warehouse should use one consistent format.

Transformation standardizes the dates.

---

# 🔤 Example 3 — Fix Text

Different systems may store:

```text
Delhi

DELHI

delhi
```

These all refer to the same city.

Transformation converts them into one consistent value.

---

# 💱 Example 4 — Currency Conversion

Imagine sales from:

🇮🇳 India → INR

🇺🇸 USA → USD

🇪🇺 Europe → EUR

If the company wants one global sales report,

currencies may need to be converted into a common unit.

---

# 🧮 Example 5 — Derived Columns

Suppose the source only stores:

```text
Date of Birth

2002-05-14
```

The warehouse may create an additional field:

```text
Age

24
```

This is a transformed value.

---

# 📥 Step 3 — Load

After transformation,

the cleaned data is stored inside the Data Warehouse.

```text

↓

↓

Data Warehouse
```

Now analysts can generate reports without affecting production systems.

# 🤔 What is ELT?

Modern cloud platforms introduced another approach.

Instead of:

```text
Extract

↓

Transform

↓

Load
```

they use:

```text
Extract

↓

Load

↓

Transform
```

This is called **ELT**.

---

# 📖 Definition

**ELT** stands for:

* **E** → Extract
* **L** → Load
* **T** → Transform

In ELT, raw data is first loaded into the warehouse.

Transformation happens **inside the warehouse**.

---

# 🏗️ ELT Architecture

```text
Source Systems

↓

Extract

↓

Load Raw Data

↓

Data Warehouse

↓

Transform Inside Warehouse

↓

Reports
```

---

# 🤔 Why Was ELT Introduced?

Modern cloud data warehouses have:

* Massive storage
* Powerful CPUs
* Parallel processing
* Distributed computing

Instead of transforming data before loading,

they load raw data first and transform it later using the warehouse's computing power.

---

# ⚖️ ETL vs ELT

| Feature             | ETL                | ELT                  |
| ------------------- | ------------------ | -------------------- |
| Transformation      | Before Loading     | After Loading        |
| Warehouse Stores    | Cleaned Data       | Raw + Processed Data |
| Processing Location | ETL Tool           | Data Warehouse       |
| Traditional Use     | On-Premise Systems | Cloud Data Platforms |
| Flexibility         | Lower              | Higher               |

---

# 🏢 Real-World Example

### Traditional Enterprise

A bank may use ETL because it wants validated, cleaned data before it enters the warehouse.

---

### Modern Cloud Company

A cloud-native company may use ELT because cloud warehouses can efficiently transform very large datasets after loading.

---

# ⚡ Batch vs Streaming

Data movement doesn't always happen the same way.

### 📦 Batch Processing

Data moves every hour or every night.

Example:

```text
Today's Sales

↓

Midnight

↓

Warehouse
```

---

### 🌊 Streaming

Data moves continuously.

```text
Order Created

↓

Immediately

↓

Warehouse
```

Streaming enables near real-time analytics.

# 🎤 FAANG Interview Questions

## ❓ What is ETL?

### ✅ Answer

ETL is the process of Extracting data from source systems, Transforming it into a clean and consistent format, and Loading it into a Data Warehouse for reporting and analytics.

---

## ❓ What is ELT?

### ✅ Answer

ELT extracts data from source systems, loads it directly into the Data Warehouse, and performs transformations afterward using the warehouse's processing capabilities.

---

## ❓ What is the main difference between ETL and ELT?

### ✅ Answer

In ETL, transformation occurs before loading the data into the warehouse.

In ELT, data is loaded first, and transformation happens inside the warehouse.

---

## ❓ Why do companies use ETL?

### ✅ Answer

ETL allows organizations to collect data from multiple sources, clean and standardize it, remove inconsistencies, and prepare it for reliable reporting and business analysis.

---

# 🎯 Key Takeaways

✅ ETL stands for Extract, Transform, Load.

✅ ELT stands for Extract, Load, Transform.

✅ ETL transforms data before loading it into the warehouse.

✅ ELT loads raw data first and transforms it later.

✅ ETL improves data quality through cleaning and standardization.

✅ ELT is common in modern cloud data platforms because warehouses provide powerful compute resources.

✅ Batch processing moves data periodically, while streaming moves data continuously.
