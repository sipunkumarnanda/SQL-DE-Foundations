# 📖 Important Definitions

## 🏛️ Relational Database

A relational database stores data in **tables** and connects those tables through **relationships** using keys.

---

## 📄 NoSQL Database

A NoSQL database stores data using models such as **documents, key-value pairs, wide columns, or graphs**, depending on the application's needs.

---

## 🔗 Relationship

A relationship is a logical connection between data stored in different tables.

---

## 📋 Table

A structured collection of related data organized into rows and columns.

---

## 📄 Row

A row (record/tuple) represents one complete record.

---

## 📌 Column

A column (field/attribute) represents one property of every record.

---

## 📦 Collection

A collection is a group of related documents in MongoDB.

---

## 📄 Document

A document is the basic unit of storage in MongoDB, typically represented as a BSON document.

---

## 🔑 Primary Key

Uniquely identifies every row in a table.

---

## 🔗 Foreign Key

References the primary key of another table to create relationships.

---

## 🆔 ObjectId

MongoDB's default unique identifier for documents.

# 🌍 SQL vs NoSQL Cheat Sheet

| Feature       | SQL                                   | NoSQL                                          |
| ------------- | ------------------------------------- | ---------------------------------------------- |
| Data Model    | Tables                                | Documents, Key-Value, Graph, Wide-Column       |
| Schema        | Fixed                                 | Flexible (in many systems)                     |
| Relationships | Strong support                        | Often embedded or application-managed          |
| Transactions  | ACID                                  | Varies by database                             |
| Joins         | Native support                        | Limited or database-dependent                  |
| Scaling       | Vertical + modern distributed options | Horizontal scaling is often a core design goal |
| Best For      | Banking, ERP, Hospitals               | Social Media, IoT, Content Platforms           |
| Examples      | MySQL, PostgreSQL                     | MongoDB, Redis, Cassandra, Neo4j               |

Semantic Validation

↓

Execution Plan

## ❓ What is a transaction?

### ✅ Answer

A transaction is a group of database operations treated as a single unit of work.

Either:

✅ Every operation succeeds.

or

❌ All operations are rolled back.

This guarantees data consistency.

# ❌ Common Mistakes

Don't say:

❌ MongoDB is better.

Say:

✅ MongoDB is better **for some workloads**.

---

Don't say:

❌ SQL is old.

Say:

✅ SQL is still the foundation of most transactional systems.

---

Don't say:

❌ NoSQL has no schema.

Say:

✅ Many NoSQL databases provide **flexible schemas**, but applications should still define and validate data structures.

---

Don't say:

❌ MongoDB replaces MySQL.

Say:

✅ MongoDB and MySQL solve different problems and are often used together.
