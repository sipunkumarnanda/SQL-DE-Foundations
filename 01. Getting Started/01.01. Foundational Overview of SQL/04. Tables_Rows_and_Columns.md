# 🌍 Introduction

Before writing SQL queries, we need to understand **how data is organized inside a database**.

Think about a coaching institute.

The institute stores information about:

* 👨‍🎓 Students
* 👨‍🏫 Teachers
* 💰 Fees
* 📅 Attendance
* 📚 Courses

Should all this information be stored in one place?

❌ No.

Instead, databases organize information into **tables**.

Everything inside a database revolves around **Tables, Rows, and Columns**.

These are the fundamental building blocks of every relational database.

---

# 🏗️ The Hierarchy of a Database

A database is organized like this:

```text
Database
│
├── 📋 Students Table
├── 💰 Fees Table
├── 📅 Attendance Table
├── 📚 Courses Table
└── 👨‍🏫 Teachers Table
```

Inside each table:

```text
Table
│
├── Columns (Vertical)
│
└── Rows (Horizontal)
```

Every piece of data belongs to a row and a column.

---

# 📋 What is a Table?

## 📖 Definition

A **Table** is a structured collection of related data organized into rows and columns.

Simply put,

> **A table stores information about one type of real-world entity.**

Each table usually represents one object.

Examples:

* 👨 Customers
* 👨 Students
* 📚 Books
* 🚗 Cars
* 💰 Orders

---

# 🏠 Real-Life Analogy

Imagine a school.

The school has different registers.

📘 Student Register

📗 Teacher Register

📕 Attendance Register

📙 Fee Register

Each register stores one type of information.

Similarly,

A database stores different information inside different tables.

---

# 💻 Example

Suppose we have an online shopping website.

The database may contain:

```text
Database
│
├── Customers
├── Products
├── Orders
├── Payments
└── Reviews
```

Notice something?

Each table stores only one type of information.

This organization makes the database easier to maintain and query.

---

# 📊 Example: Customers Table

| customer_id | name        | email                                               | signup_date | is_active |
| ----------: | ----------- | --------------------------------------------------- | ----------- | --------- |
|           1 | Aisha Khan  | [aisha@example.com](mailto:aisha@example.com)       | 2024-01-10  | 1         |

|           3 | Meera Joshi | [meera.j@example.com](mailto:meera.j@example.com)   | 2024-03-01  | 1         |

This table stores only customer information.

It does **not** store:

❌ Product Details

❌ Orders

❌ Payments

Those belong in separate tables.

---

# 🤔 Why Do We Use Separate Tables?

Imagine storing everything in one giant table.

```text
Student
Teacher
Salary
Attendance
Books
Fees
Payments
Courses
```

Very quickly,

it becomes confusing.

Data gets duplicated.

Maintenance becomes difficult.

Searching becomes slower.

Instead,

we divide information into multiple related tables.

This improves:

✅ Organization

✅ Readability

✅ Performance

✅ Data Integrity

---

# 📌 What is a Row?

## 📖 Definition

A **Row** (also called a **Record** or **Tuple**) represents one complete entry inside a table.

Every horizontal line in a table is one row.

Each row describes one real-world object.

---

# 🧑 Real-Life Analogy

Imagine a school admission register.

Each student fills one admission form.

One form contains:

* Name
* Phone
* Address
* Email
* Course

That entire form represents **one student**.

Similarly,

one row represents one complete record.

---

# 💻 Example

Students Table

| Student ID | Name        | Email                                               | City   |
| ---------- | ----------- | --------------------------------------------------- | ------ |
| 1          | Rahul Mehta | [rahul.mehta@mail.com](mailto:rahul.mehta@mail.com) | Mumbai |

This single horizontal line represents one student.

Everything about Rahul is stored in one row.

---

# 📝 Different Names for a Row

A row is also called:

✅ Record

# 📌 What is a Column?

## 📖 Definition

A **Column** (also called a **Field** or **Attribute**) represents one specific property or characteristic of the data.

Every vertical section in a table is one column.

---

# 🏠 Real-Life Analogy

Think about your Aadhaar card.

It contains different pieces of information:

* Name
* Date of Birth
* Address
* Gender
* Aadhaar Number

Each of these is an **attribute**.

Similarly,

each column stores one attribute about every record.

---

# 💻 Example

Students Table

| Student ID | Name  | Email                                     | City   |
| ---------- | ----- | ----------------------------------------- | ------ |
| 1          | Rahul | [rahul@gmail.com](mailto:rahul@gmail.com) | Mumbai |

Columns are:

📌 Student ID

📌 Name

📌 Email

📌 City

Each column stores only one type of information.

# ❓ Common Beginner Confusion

### Is a Table the Same as a Database?

❌ No.

A database contains many tables.

Example:

```text
Amazon Database

│

├── Customers

├── Orders

├── Products

├── Sellers

├── Reviews
```

One database can contain hundreds or even thousands of tables.

---

### Does Every Row Have the Same Columns?

✅ Yes.

Every row in a table follows the same structure.

Example:

| ID | Name  | Email                                     |
| -- | ----- | ----------------------------------------- |
| 1  | Rahul | [rahul@gmail.com](mailto:rahul@gmail.com) |
| 2  | Priya | [priya@gmail.com](mailto:priya@gmail.com) |

Every row has:

* ID
* Name
* Email

This consistency makes querying and processing data much easier.

# 🎤 Interview Questions

## ❓ What is a Table?

**Answer:**

A table is a structured collection of related data organized into rows and columns. It represents one type of real-world entity, such as students, customers, or orders.

---

## ❓ What is a Row?

**Answer:**

A row (also called a record or tuple) represents one complete record in a table. Each row stores all the information related to a single entity.

---

## ❓ What is a Column?

**Answer:**

A column (also called a field or attribute) represents a specific property of the data stored in a table. All values in a column are of the same type or represent the same kind of information.

---

## ❓ Why Do Databases Use Multiple Tables?

**Answer:**

Databases use multiple tables to organize related data efficiently, reduce redundancy, improve maintainability, ensure data integrity, and simplify querying.

---

# 🎯 Key Takeaways

✅ A **Database** contains one or more **Tables**.

✅ A **Table** stores data about one specific type of entity.

✅ A **Row** (Record/Tuple) represents one complete record.

✅ A **Column** (Field/Attribute) represents one property of every record.

✅ Every row has the same set of columns.

✅ Organizing data into separate tables improves scalability, consistency, and performance.
