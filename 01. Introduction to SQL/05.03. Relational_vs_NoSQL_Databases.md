# 🧠 What is a Relationship?

## 📖 Definition

A **Relationship** is a logical connection between two or more tables.

It allows the database to understand

how data stored in different tables is related.

Simply put,

> A relationship connects related information stored in separate tables.

---

The answer exists because

Rahul and his father have a relationship.

Without relationships,

they would just be two unrelated people.

# 🔑 The Secret Behind Relationships

Relationships are usually created using **Keys**.

There are two important concepts:

* 🔑 Primary Key
* 🔗 Foreign Key

We will study these in depth in later chapters.

For now, think of them like this.

---

# 🔑 Primary Key (Introduction)

A Primary Key uniquely identifies every row in a table.

Example

Students Table

| Student ID | Name  |
| ---------- | ----- |
| 101        | Rahul |
| 102        | Priya |
| 103        | Ankit |

Student ID is unique.

No two students can have the same Student ID.

---

# 🔗 Foreign Key (Introduction)

A Foreign Key stores the Primary Key value of another table.

Attendance Table

| Attendance ID | Student ID | Date   |
| ------------- | ---------- | ------ |
| 1             | 101        | 1 July |
| 2             | 101        | 2 July |
| 3             | 102        | 1 July |

Student ID here refers back to the Students table.

That creates the relationship.

---

# 🎯 Types of Relationships

There are three major relationship types.

```text
Database Relationships

│

├── One-to-One (1 : 1)

├── One-to-Many (1 : N)

└── Many-to-Many (M : N)
```

These three relationships can model almost every real-world system.

---

# 1️⃣ One-to-One Relationship (1:1)

## 📖 Definition

One record in Table A is related to only one record in Table B.

Likewise,

one record in Table B belongs to only one record in Table A.

---

Passport

This is the **most common relationship** in relational databases.

## 📖 Definition

But each record in Table B belongs to only one record in Table A.

# 🤔 The Problem

Can we directly connect

Students

and

Courses?

Not efficiently.

many students choose many courses.

---

# 💡 The Solution

We introduce a third table called a **Junction Table** (also called a Bridge or Mapping Table).

```text
Students

↓

Enrollments

↓

Courses
```

---

Students Table

| Student ID | Name  |
| ---------- | ----- |
| 101        | Rahul |
| 102        | Priya |

Courses Table

| Course ID | Course |
| --------- | ------ |
| 10        | Java   |
| 20        | SQL    |

Enrollments Table

| Student ID | Course ID |
| ---------- | --------- |
| 101        | 10        |
| 101        | 20        |
| 102        | 20        |

Now,

Rahul studies Java and SQL.

Priya studies SQL.

This models the many-to-many relationship cleanly.

# 🎤 FAANG Interview Questions

## ❓ What are the three types of database relationships?

### ✅ Answer

The three primary relationship types are:

* **One-to-One (1:1):** One record is associated with exactly one record in another table.
* **One-to-Many (1:N):** One record is associated with multiple records in another table, while each of those records belongs to only one parent.
* **Many-to-Many (M:N):** Multiple records in one table are associated with multiple records in another table. This is typically implemented using a junction table.

---

## ❓ Why can't databases directly implement Many-to-Many relationships?

### ✅ Answer

Because relational databases represent relationships through keys. A direct many-to-many connection would require storing multiple values in a single column, violating the principles of the relational model. Instead, an intermediate junction table is used to store pairs of related keys, allowing the relationship to be represented correctly.

# 🎯 Key Takeaways

✅ Relationships connect related data across tables.

✅ Primary Keys uniquely identify records.

✅ Foreign Keys reference records in other tables.

✅ Many-to-Many relationships are implemented using a junction table.
